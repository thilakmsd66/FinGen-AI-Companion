OSI Poststrasse: In the process of becoming global leader in Security Custody business State Street Corporation acquired subsidiary of Deutsch Bank AG, which operates in security custody business in German market.  Because of this acquisition, State Street Corporation is now integrating its systems with Deutsche Bank AG (renamed as State Street GmbH after acquisition). Poststrasse is one of the systems of State Street GmbH that sends requested reports in AFP / SWIFT format to DB Print for printing. The printing facility of DB Print prints these reports and manually posts to their clients at their mailing address. This is a critical application since it is open for the clients and any small mistake can be indicated back to State Street.
At a high level, the Poststrasse application consists of a Front End Application, and a Back End Application.
The Poststrasse Front End application is an online application with a web client that allows its users to deal with Link (Unique combination of Fund, Address and Report Type) and Queue (Links in processing state). Whenever any user wants any specific report of a particular fund to be mailed to his/her address, he/she can create a Link using the requested Fund, Report type and his/her address. User can view the status of execution of any report type; he/she has requested for and can also update the schedules of his/her report as per their requirements. 
The Poststrasse Back End application is a set of daily, monthly, quarterly and yearly batch processing programs that executes as per the schedule set by the users. Its users schedule the reports (using PS Front end) at any specified time and back end invokes different handlers to execute it at the scheduled time.
